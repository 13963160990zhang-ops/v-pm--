基于参考UI风格，长图需遵循以下设计标准：

### 整体风格
- **双模式支持**：默认深色模式（背景 RGB 11,23,37 深蓝黑色），可选浅色模式（背景米白/纯白）
- **圆角卡片**：所有数据模块采用圆角矩形卡片，带轻微阴影或边框分隔
- **留白节奏**：模块间保持充足间距，避免信息拥挤

### 配色方案
| 元素 | 深色模式 | 浅色模式 |
|------|----------|----------|
| 背景 | #0B1725 深蓝黑 | #FEFCFB 米白 |
| 卡片背景 | #0C1521 略浅于背景 | #FFFFFF 纯白 |
| 主文字 | #FFFFFF 白 | #1A1A1A 深黑 |
| 次要文字 | #8B9AAF 灰蓝 | #666666 灰 |
| 强调色 | #4A90D9 蓝 / #45B7D1 青 | 同深色模式 |
| 正增长 | #4CAF50 绿 | #4CAF50 绿 |
| 负增长 | #EF5350 红 | #EF5350 红 |

### 布局结构（自上而下）
1. **顶部标题栏**：日期标识 + 报告名称（如"快游戏日报-2026-06-08"）
2. **核心指标区**：3-4个大号数字卡片，展示昨日关键数据（消耗金额、CTR、曝光量等）
3. **趋势图表区**：折线图/面积图，时间轴展示近期走势，支持正负区间显示
4. **占比分析区**：饼图或横向柱状图，展示广告位类型/渠道占比
5. **异动说明区**：文字卡片，列出TOP3异动及原因分析
6. **底部汇总**：数据来源说明 + 生成时间戳

### 字体层级
| 层级 | 用途 | 字号建议 |
|------|------|----------|
| H1 | 报告标题 | 36-40px |
| H2 | 模块标题 | 24-28px |
| 数据指标 | 核心数字 | 48-56px 加粗 |
| 单位/标签 | 指标说明 | 14-16px |
| 正文 | 分析结论 | 16-18px |

### 图表规范
- **折线图**：带渐变填充的面积图，线条 2-3px，数据点明显
- **柱状图**：圆角柱体，间距为柱宽 1/3
- **百分比**：环形图或横向进度条，标注具体数值
- **坐标轴**：浅色细线，刻度标签灰蓝色

### 模块间距
- 模块间垂直间距：40-60px
- 卡片内边距：24-32px
- 卡片圆角：12-16px

## 执行流程

### 每日分析任务总览
按以下顺序逐项执行，每项任务需先读取对应reference了解详细需求：
1. **快游戏流量日报** → 读取 references/quickgame.md
2. **单机游戏流量日报** → 读取 references/apkgame.md
3. **每日分析任务3** → 待补充

### 每个任务的标准执行模式
1. 读取对应reference，了解数据源、分析步骤、产出要求
2. 通过 opencli bivivo download 下载数据，下载后为Excel/CSV文件
   - **下载范围**：仅下载截止到昨天且本地尚缺失的日期数据。例如：本地已有至6月1日的数据，今天是6月5日，则只下载补充6月2日-6月4日的数据，避免重复
3. 将下载的数据追加到本地数据基底Excel，不要覆盖历史数据
4. 基于本地数据基底Excel按分析思路处理数据，产出关键结论
   - **分析主体**：实际数据分析（如消耗金额、分广告位类型占比等指标）仅以**昨天**单日数据为主体。例如：今天是6月5日，则只分析6月4日的数据
5. 汇总所有任务产出，生成最终可视化长图
6. 保存长图至指定目录，命名按格式
7. 推送至V消息：
   - 上传长图至V消息文件服务器
   - 发送IMAGE类型群聊消息（长图）
   - 发送TEXT类型群聊消息（关键数据摘要文字）
